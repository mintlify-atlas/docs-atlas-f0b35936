## Why Antiox?

Antiox exists for a specific niche: TypeScript codebases that want Rust-shaped concurrency without the weight of a full effect system.

<CardGroup cols={2}>
  <Card title="Channels & backpressure" icon="arrows-left-right">
    Bounded MPSC, oneshot, broadcast, and watch channels with clean shutdown via `for await` iteration
  </Card>
  <Card title="Async streams" icon="wave-sine">
    Composable stream combinators on native `AsyncIterable&lt;T&gt;` — no wrapper objects
  </Card>
  <Card title="Task management" icon="list-check">
    Spawn tasks with `AbortSignal` cancellation, manage groups with `JoinSet`
  </Card>
  <Card title="Sync primitives" icon="lock">
    Mutex, RwLock, Semaphore, Barrier, Notify — everything you need to coordinate async tasks
  </Card>
  <Card title="Zero dependencies" icon="feather">
    No transitive bloat. Every module is tree-shakeable and tiny enough to ship inside a library
  </Card>
  <Card title="Rust-shaped API" icon="gear">
    Same naming and semantics as Tokio — your TypeScript reads like the Rust it was ported from
  </Card>
</CardGroup>

## Getting started

<Steps>
  <Step title="Install the package">
    ```bash
    npm install antiox
    ```
  </Step>
  <Step title="Import the modules you need">
    Each primitive lives in its own subpath — import only what you use:
    ```typescript
    import { channel } from "antiox/sync/mpsc";
    import { spawn } from "antiox/task";
    import { sleep } from "antiox/time";
    ```
  </Step>
  <Step title="Build your first actor">
    Use a bounded channel and `for await` to process messages with backpressure:
    ```typescript
    import { channel } from "antiox/sync/mpsc";
    import { spawn } from "antiox/task";

    const [tx, rx] = channel<string>(32);

    spawn(async () => {
      for await (const msg of rx) {
        console.log("received:", msg);
      }
    });

    await tx.send("hello");
    await tx.send("world");
    tx.close();
    ```
  </Step>
  <Step title="Explore the full API">
    Browse the [API Reference](/api/mpsc) for every module, or read the [Concepts](/concepts/channels-and-backpressure) to understand the design.
  </Step>
</Steps>

<Note>
  Antiox is pre-release. The library is used in production, but the API is subject to change before 1.0.
</Note>
