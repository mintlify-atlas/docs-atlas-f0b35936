---
title: "Quickstart"
description: "Build a typed counter actor using channels and oneshot."
---

This guide walks through the canonical Antiox example: a counter actor that receives typed messages over an MPSC channel and responds to queries via oneshot channels. It demonstrates the core primitives you will use in almost every Antiox-based system.

<Steps>

<Step title="Install Antiox">

If you have not installed Antiox yet, add it to your project:

```bash
npm install antiox
```

</Step>

<Step title="Define your message type">

Antiox channels are fully typed. Start by defining a union type for every message your actor will handle:

```typescript
import { OneshotSender } from "antiox/sync/oneshot";

type Msg =
  | { type: "increment"; amount: number }
  | { type: "get"; resTx: OneshotSender<number> };
```

`increment` is a fire-and-forget message. `get` carries a oneshot sender so the caller can receive a typed response. Using a discriminated union means the compiler enforces exhaustiveness when you handle messages.

</Step>

<Step title="Create the channel">

Create a bounded MPSC channel with a capacity of 32 messages:

```typescript
import { channel } from "antiox/sync/mpsc";

const [tx, rx] = channel<Msg>(32);
```

`tx` is the sender. `rx` is the receiver. Only one receiver can exist at a time (single-consumer), but you can clone `tx` to create additional senders.

The capacity of 32 provides backpressure: senders that call `tx.send()` will await when the buffer is full, preventing unbounded memory growth.

</Step>

<Step title="Spawn the actor">

Spawn a background task that owns the receiver and processes messages in a `for await` loop:

```typescript
import { spawn } from "antiox/task";
import { unreachable } from "antiox/panic";

spawn(async () => {
  let count = 0;
  for await (const msg of rx) {
    switch (msg.type) {
      case "increment":
        count += msg.amount;
        break;
      case "get":
        msg.resTx.send(count);
        break;
      default:
        unreachable(msg);
    }
  }
});
```

`spawn` runs the function on the next microtask. It returns a `JoinHandle` you can `await` or `.abort()`. The `for await` loop exits cleanly when all senders are dropped, so shutdown is automatic.

`unreachable(msg)` in the `default` branch is a compile-time exhaustiveness check. If you add a new variant to `Msg` without handling it, TypeScript will error because the unhandled variant is not assignable to `never`.

</Step>

<Step title="Send messages and query state">

Send fire-and-forget messages with `tx.send()`. For request-response, create a oneshot channel, embed the sender in the message, and await the receiver:

```typescript
import { oneshot } from "antiox/sync/oneshot";

// Fire-and-forget
await tx.send({ type: "increment", amount: 5 });

// Request-response via oneshot channel
const [resTx, resRx] = oneshot<number>();
await tx.send({ type: "get", resTx });
const value = await resRx;
```

`oneshot()` returns a `[OneshotSender, OneshotReceiver]` pair. The sender can transmit exactly one value; the receiver implements `PromiseLike<T>`, so you can `await` it directly. There are no locks or shared mutable state — the count lives entirely inside the actor's closure.

</Step>

<Step title="Complete example">

Putting it all together:

```typescript
import { channel } from "antiox/sync/mpsc";
import { oneshot, OneshotSender } from "antiox/sync/oneshot";
import { unreachable } from "antiox/panic";
import { spawn } from "antiox/task";

type Msg =
  | { type: "increment"; amount: number }
  | { type: "get"; resTx: OneshotSender<number> };

const [tx, rx] = channel<Msg>(32);

spawn(async () => {
  let count = 0;
  for await (const msg of rx) {
    switch (msg.type) {
      case "increment":
        count += msg.amount;
        break;
      case "get":
        msg.resTx.send(count);
        break;
      default:
        unreachable(msg);
    }
  }
});

// Fire-and-forget
await tx.send({ type: "increment", amount: 5 });

// Request-response via oneshot channel
const [resTx, resRx] = oneshot<number>();
await tx.send({ type: "get", resTx });
const value = await resRx;
```

</Step>

</Steps>
